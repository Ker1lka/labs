## 🛠️ Опис реалізації CI/CD процесів

### 1. Перевірка коду (CI)
*   **Файл конфігурації:** `ci.yml`
*   **Тригери запуску:** Події `push` та `pull_request` у головну гілку `main`.
*   **Опис роботи:** 
    *   Розгортає віртуальне середовище на базі `ubuntu-latest`.
    *   Встановлює стабільну версію Node.js.
    *   Проводить чисте встановлення пакетів за допомогою `npm ci`.
    *   Запускає лінтер (`npm run lint`), тести (`npm test`) та тестову збірку проєкту (`npm run build`). Це гарантує, що зламаний код не потрапить у продакшн.

### 2. Автоматичний деплой (CD)
*   **Файл конфігурації:** `deploy.yml`
*   **Тригери запуску:** Успішний `push` або `pull_request` в гілку `main`.
*   **Опис роботи:** 
    *   Використовує офіційні екшени GitHub для конфігурації Pages.
    *   Компілює React/Vite проєкт, створюючи оптимізовану збірку у папці `dist`.
    *   Завантажує вміст папки `dist` як артефакт і безпечно публікує його на GitHub Pages.
    *   Налаштовано механізм `concurrency`, що скасовує старі деплої, якщо паралельно запустився новий, уникаючи конфліктів перевизначення сайту.

### 3. Автоматизація релізів (Release Management)
*   **Файл конфігурації:** `release.yml`
*   **Тригери запуску:** Автоматично запускається при оновленні гілки `main`.
*   **Опис роботи (Інтеграція з API):**
    *   Робот робить HTTP-запит до публічного API `https://api.chucknorris.io/`.
    *   За допомогою утиліти `jq` парсить отриманий JSON-об'єкт та витягує випадковий жарт.
    *   Використовуючи утиліту `sed`, динамічно оновлює локальний файл `README.md` у репозиторії, дописуючи туди інформацію про нову версію, назву коміту та згенерований жарт.
    *   Робить автоматичний коміт змін із тегом `[skip ci]`, щоб запобігти нескінченному циклу викликів Actions.
    *   Викликає екшен `create-release`, створюючи офіційний реліз із унікальним тегом виду `v1.0.{github.run_number}` та додає отриманий жарт в опис релізу.
